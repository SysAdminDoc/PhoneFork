# PhoneFork AI Logo And Icon Prompts

Use these prompts to generate logo and icon directions for PhoneFork, a Windows desktop tool for local Android phone migration. The visual identity should feel premium, technical, trustworthy, minimal, and practical.

## Shared Brand Direction

Product signals to preserve:
- Name: PhoneFork
- Category: Windows desktop Android migration utility
- Core idea: move data between two phones locally, without cloud, account lock-in, or root
- Audience: sysadmins, power users, privacy-conscious Android users
- Mood: calm, precise, premium, local-first, reliable
- UI palette: Catppuccin Mocha style dark base, mauve/lavender/blue accent, soft green readiness signal

Avoid:
- Samsung, Android, Google, Play Store, USB, or Windows official logos
- Literal cloud imagery
- Busy circuit-board decoration
- Mascots, cartoon characters, glossy 3D objects, generic app-store clip art
- Tiny text inside the icon
- Overly cute rounded-bubble styling

Preferred output requirements:
- 1:1 app icon, 512x512 or larger
- Transparent PNG or clean vector-style SVG when possible
- Works at 16x16, 32x32, 48x48, 256x256, and 512x512
- High contrast on both dark and light backgrounds
- No text in the app icon unless generating the wordmark prompt

## Prompt 1: Minimal App Icon

Create a premium minimal app icon for "PhoneFork", a local-first Android phone migration tool for Windows. The mark should show two simplified phone silhouettes connected by a subtle forked transfer path, suggesting source and destination without using arrows as the main idea. Use a dark Catppuccin Mocha-inspired base with a refined mauve-to-blue accent and a small soft green readiness detail. The icon must feel calm, trustworthy, technical, and modern, with crisp geometry, strong negative space, and excellent legibility at small sizes. No text, no official Android/Samsung/Google/USB logos, no cloud imagery, no mascot, no glossy 3D render. Transparent background, vector-style, 512x512.

## Prompt 2: Premium Windows App Icon

Design a polished Windows desktop app icon for "PhoneFork". The concept is a secure local bridge between two Android phones: two abstract device panels, one source and one destination, connected by a clean branching cable/fork motif. Use a deep charcoal/navy background shape, Catppuccin-inspired mauve and lavender highlights, restrained blue accents, and one small green status point to imply trusted connected devices. The icon should look professional beside other Windows utilities, with balanced spacing, sharp edges, subtle depth, and no visual clutter. Avoid brand logos, clouds, lock cliches, QR codes, text, cartoon style, and overly rounded pill shapes. Transparent PNG, vector-like, readable at 32x32.

## Prompt 3: Trust And Migration Emblem

Create a compact emblem for PhoneFork, a no-root, no-cloud phone migration and debloat utility. Represent "local transfer you can audit" through an abstract forked path enclosed in a precise geometric frame. Include two small device nodes and a branching path that feels intentional, not like a generic sync arrow. Visual tone: premium system utility, privacy-first, precise, calm, dark-mode native. Palette: Catppuccin Mocha base, mauve/lavender primary accent, blue secondary accent, soft green confirmation detail. Flat vector style with subtle layered surfaces, crisp contrast, no text, no cloud, no official platform marks, no shield unless highly abstract. Transparent background, square icon.

## Prompt 4: Wordmark Plus Symbol

Create a refined wordmark and symbol lockup for "PhoneFork". The symbol should be a simple two-phone forked-transfer mark that can stand alone as an app icon. The wordmark should use clean modern sans-serif lettering, balanced spacing, and a technical but approachable tone. The full lockup should work on a dark Catppuccin Mocha background and in monochrome. Color accents should use mauve/lavender with a restrained blue secondary. Avoid futuristic sci-fi fonts, gamer styling, corporate cloud imagery, official Android/Samsung/Google marks, and tiny decorative detail. Provide horizontal and stacked compositions, with a separate icon-only version.

## Prompt 5: GitHub Banner / README Hero

Create a tasteful GitHub README banner for PhoneFork, a Windows tool that migrates apps, media, settings, Wi-Fi, roles, and debloat profiles between two Samsung phones locally. The banner should feature the PhoneFork icon mark on the left and an elegant dark product-utility composition on the right: two abstract phones, a local transfer path, small audit-log/status details, and no cloud. Use a Catppuccin Mocha-inspired palette with mauve/lavender primary accents, blue secondary accents, and soft green readiness indicators. The layout should feel premium, minimal, privacy-conscious, and trustworthy. Avoid screenshots of fake UI, official phone brand logos, messy cables, mascots, neon cyberpunk, and excessive gradients. Wide banner, transparent or dark background, clean vector-style finish.

## Universal Negative Prompt

Do not include official Android, Samsung, Google, Play Store, USB, or Windows logos. Do not include clouds, cloud upload/download symbols, padlock cliches, mascots, cartoon faces, glossy 3D phone renders, cluttered circuit boards, QR codes, tiny unreadable text, cheap stock icon style, excessive gradients, lens flares, or generic sync arrows. Keep the mark crisp, minimal, high-contrast, premium, and readable at small sizes.

## Selection Criteria

Choose the strongest result by these checks:
- Recognizable at 32x32 without relying on text
- Clearly suggests two-device local migration
- Feels trustworthy and technical without looking cold or corporate
- Avoids platform trademark risk
- Matches the current dark Catppuccin-inspired UI
- Can become `.ico`, 512x512 PNG, README banner, and app title symbol
